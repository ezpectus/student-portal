### Security fixes
- [x] `requestPasswordReset` тАФ removed `resetToken` from API response. Token now stored in notification message as reset link. Dead code in `auth.actions.ts` that checked for `resetToken` in response also removed.
- [x] `resetPassword` тАФ `z.string().min(8).max(128)` validation already present (audit was incorrect)
- [x] `getGradeHistory` тАФ ownership check already present: `isOwner || isTeacher || isAdmin` + `schoolId` filter (audit was incorrect)
- [x] AI chat `message` тАФ `z.string().min(1).max(500)` validation already present (audit was incorrect)

### Build fixes (critical тАФ production build was broken)
- [x] **Edge Runtime crypto** тАФ `middleware.ts` used Node.js `crypto.randomBytes` (not supported in Edge Runtime). Replaced with Web Crypto API `crypto.getRandomValues` + `btoa`.
- [x] **csrf.ts Edge Runtime** тАФ Split `generateCsrfToken` and `validateCsrfToken` into `csrf-utils.ts` (Edge-safe, no `server-only` import). `csrf.ts` kept for `requireCsrf` (server actions only). Replaced `createHash` with constant-time char comparison.
- [x] **'use server' re-exports** тАФ `local-auth.actions.ts` re-exported from `local-password.actions.ts` and `local-user.actions.ts`. Turbopack cannot resolve these. Removed re-exports, updated all 20+ importers to import directly from source files.
- [x] **'use server' non-async export** тАФ `local-user.actions.ts` exported `getModulesForRole` (non-async function) which violates Next.js 15 `'use server'` rules. Moved to `src/lib/get-modules-for-role.ts` (plain module). Updated all 3 importers.
- [x] **Missing i18n keys** тАФ `login-carousel.tsx` uses `useTranslations('public.cards.carousel')` but `public.cards.carousel` didn't exist in any locale file (only `private.main.cards.carousel` existed). Added `public.cards.carousel` to all 4 locale files (en, uk, pl, de).

### Verification
- [x] `calendar.actions.ts` тАФ `Event` model exists in Prisma schema with all required fields (title, description, startDate, endDate, location, type, schoolId, createdBy)
- [x] `twoFactor*` fields already removed from Prisma schema (audit was incorrect)

- [x] **menu.actions.ts shadowing** тАФ Renamed destructured `group` variable to `groupName` to avoid shadowing imported `radash.group()` function.
